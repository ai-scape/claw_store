## PDF Generation — Root Cause & Fix

**Root cause of import failures:** When `make_pdf.py` runs from the `reports/` directory, `sys.path[0]` = `reports/` and `sys.path[1]` = `reports/`'s parent `ai-intel/`. The `ai-intel/markdown/` subdirectory (created during the session) shadows the real `markdown` package in `Miniconda3\Lib\site-packages\`, and `reports/markdown/` shadows it too.

**Fix in `reports/make_pdf.py`:** Remove both directories from `sys.path` before importing, then insert Miniconda site-packages at index 1 so real packages are found first:
```python
_miniconda_site = r"C:\Users\Welcome\Miniconda3\Lib\site-packages"
if _miniconda_site not in sys.path:
    sys.path.insert(1, _miniconda_site)
for _dir in [os.path.dirname(os.path.abspath(__file__)),
             os.path.dirname(os.path.abspath(__file__)).rsplit(os.sep, 1)[0]]:
    if _dir in sys.path:
        sys.path.remove(_dir)
```
Run with: `C:\Users\Welcome\Miniconda3\python.exe C:\Users\Welcome\.openclaw\workspace\ai-intel\reports\make_pdf.py`
